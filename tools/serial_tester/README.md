# Servo Serial Tester

Python tkinter GUI，用來測試 Arduino Nano + Servo 行為。  
不需要安裝 PyQt6，只需要 Python 內建的 tkinter + pyserial。

---

## 啟動

```bash
pip install pyserial
python serial_tester.py
```

---

│ └──────────────────────┘  │ 14:02:01  ← OK PONG                  │ │
│                            │ 14:02:05  → BEGIN 2                  │ │
│ ┌── 單步測試 ──────────┐  │ 14:02:05  ← OK RECEIVING             │ │

│ │ 角度°   [   90]  ◜   │  ├──────────────────────────────────────┤ │

│ │ 停留ms  [  300]       │  └──────────────────────────────────────┘ │

### 快速指令區
- **PING**：確認 Arduino 是否回應（應回 `OK PONG`）
- **STATUS**：查詢目前狀態
- **STOP**：緊急停止並歸位

### 單步測試區
- 設定延遲、角度、速度、停留參數
- 角度指示器（小圓弧）即時預覽目標角度
- **送出單步**：自動包成 BEGIN/STEP/END 腳本送出
- **預覽指令**：在 Log 顯示會送出的指令，不實際送出

### 腳本測試區
- 直接編輯 `LOOP / BEGIN / STEP / END` 原始指令
- 可貼入任意腳本內容
- 勾選「循環」會自動覆蓋 LOOP 設定
- **# 開頭的行會被忽略**（當作註解）

### Serial Log 區
- 顏色分類：
  - 🔵 藍色：送出的指令
  - 🟢 綠色：OK READY / OK DONE / OK PONG
  - 🟡 琥珀色：OK RUNNING（執行中）
  - 🔴 紅色：ERR 錯誤
- 手動輸入任意指令（按 Enter 送出）

## 常見問題

**Q: 找不到 COM port？**  
A: 安裝 CH340 驅動：https://www.wch.cn/downloads/CH341SER_EXE.html  
插拔 USB 後點 ⟳ 刷新。

**Q: 連線成功但 Servo 沒動？**  
1. 先送 `PING` 確認回應
2. 再送 `STATUS` 確認狀態是 `OK IDLE`
3. 試試縮短 delay_ms（例如改成 500）確認 Servo 是否會動
4. 確認接線：D9 訊號、5V 電源、GND 共地

**Q: ERR BUSY 錯誤？**  
A: 正在執行中，先送 `STOP` 再重試。

**Q: 送出指令後沒有任何回應？**  
A: 鮑率不對。Arduino 設定為 115200，確認 Arduino IDE 燒錄時 `SERIAL_BAUD` 也是 115200。
